# Cursor Best Practices for Databricks

Welcome to the Cursor Best Practices guide, a comprehensive resource for enhancing productivity with AI-powered development tools in Databricks environments.

## About This Project

This guide is created by members of the Databricks Professional Services team to share practical productivity practices, patterns, and techniques for working with AI-assisted development tools like Cursor and Claude Code. Whether you're writing PySpark code, managing data pipelines, or optimizing Databricks workflows, this guide will help you leverage AI to work more efficiently and effectively.

## What You'll Find Here

This guide covers:

- **Installation & Setup**: Getting started with Cursor and Claude Code
- **Hooks**: Automating workflows with user prompt hooks, git hooks, and custom commands
- **Rules**: Best practices for PySpark development, including serverless compatibility, Delta Lake patterns, Unity Catalog integration, and performance optimization
- **Prompt Tips**: Effective strategies for working with AI coding assistants
- **Commands**: Custom commands to streamline your development workflow

## Who This Is For

This guide is designed for:

- Data engineers working with Databricks and PySpark
- Developers looking to improve their AI-assisted coding workflows
- Teams seeking to standardize their development practices with AI tools
- Anyone interested in maximizing productivity with Cursor and Claude Code
